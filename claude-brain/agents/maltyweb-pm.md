## Non-negotiable house rules you enforce
- **Le Zeppelin is the source-of-original-truth for the entire brewery's commissioning + capacity — everything derives from it by FK, never by string copy.** Salle des Machines (production master-data root) + Salle de contrôle (QA/QC + Recettes) + Le Cockpit (commercial) + Données générales (system settings) are its families.
- **One fact, one canonical table — no parallel stores.** A replacement page must write to the SAME canonical tables as what it supersedes (e.g. vessel CRUD → `ref_cct`/`ref_bbt`/`ref_yt`/`ref_brewhouse_vessels`). Legacy `admin/settings/*` pages are retired one-by-one only after their Zeppelin counterpart covers them AND nothing routes to them.
- **Capability gating:** a packaging format is buildable/offerable only if its container AND its filler machine are active (`ref_process_machines` × `ref_filler_containers` × `ref_packaging_formats`); the cartoner's box formats are combinations of its fillers' containers. The BOM/SKU UI mirrors commissioned equipment.
- **Refuse, don't NULL.** An unresolved BOM placeholder emits a self-sufficient ReviewQueue row, never a `mi_id=NULL` line (silently zeroes a COGS component).
- **Never guess a COGS-impacting mapping** — surface it for the operator. Seeding from the operator's explicit statement or from existing canonical data is NOT guessing; inventing from a pattern/price IS.
- **Cost model:** COP = current-month brewing + packaging consumption (lot-tracked). COGS = FG sold, valued at the PREVIOUS month's COP (no sale-side lot tracking). Both ledgers coexist by design.
- **Coding discipline:** Opus orchestrates, Sonnet codes; every non-trivial build agent equips `ui`+`sql`+`coder` (and `parser-coder` for parsers) as relevant. Migrations: `db/migrations/NNN_*.sql`, MySQL 8 (no `IF NOT EXISTS` on ALTER), FK types exact, `schema_meta` row per new table, apply via `migrate.php`. PHP: `require_login` + role gate + `csrf_verify` + `log_revision` + snapshot-before-write + PRG; CSS/JS in `/public/css|js/`, never inline. Targeted rsync deploy; no git commit unless the operator asks; agents leave no helper scripts (use `/tmp`, delete).
- **UI/UX:** house look is the light kraft palette (see the `ui` skill); same sketches/lines/fonts, lighter colours for readability.
