## SCOTCH MODEL (§8.1 — operator-corrected reference convention; the build had it wrong) — applies to BUCKET 1 (the 24-box)
**Scotch belongs to the 24-box format ONLY** — 24-box BOTTLE (`B`, format_id=1), 24-box CAN (`C`, format_id=7), and the `BC` variant (format_id=8). NOT "all bottle recipes". For a 24-box, scotch is ONE OF two MUTUALLY-EXCLUSIVE configs (per recipe):
- **(A)** branded `PKG_SCOTCH_[BEER]` alone; OR
- **(B)** `PKG_SCOTCH_TRANSP` (transparent box-tape) **+** `PKG_STICKER_[BEER]` (a beer sticker on the box).

**VERIFIED MIs (ref_mi): scotch family** id176 PKG_SCOTCH_DIV, 177 PKG_SCOTCH_EMB, 178 PKG_SCOTCH_MOO, **179 PKG_SCOTCH_TRANSP** (slot default), 180 PKG_SCOTCH_ZEP. **sticker family:** 181 PKG_STICKER_ALT, 182 PKG_STICKER_DIB_DK, 271 PKG_STICKER_EMB, 183 PKG_STICKER_SPY_DK, 184 PKG_STICKER_STI, 185 PKG_STICKER_ZEPC. (Branded scotch exists only for DIV/EMB/MOO/ZEP; **every other beer can ONLY do config B = TRANSP+sticker**.) `uses_branded_scotch` EXISTS on `ref_recipes`: =1 for DIV/EMB/MOO/ZEP, =0 for the 16 others (authoritative recipe-level A/B signal if auto-suggesting; but the binding is the SoT the recompute reads).

**⚠️ uses_branded_scotch is RECIPE-altitude — known limitation (incident 2026-06-07, step-② batch):** branded scotch is a BOTTLE-box thing, but the flag is per-recipe, so ZEP's =1 leaked onto the CAN box (ZEPC) — branded scotch applied + §8.1 auto-suppression killed ZEPC's sticker. **Fixed via Tier-1 per-SKU choice id 47 (sku 59 ZEPC, slot scotch → PKG_SCOTCH_TRANSP id179) → PKG_STICKER_ZEPC ×1 restored.** Today the per-SKU choice covers the ONLY -C of a branded-scotch beer; **if a 2nd -C (or any non-bottle box) of DIV/EMB/MOO/ZEP ever ships, make the flag FORMAT-scoped (bottles-only) instead of multiplying choices.** Related intentional oddity: BLO4C (inactive) still carries a Nomoq intercal row — historical-only per the BLA/BLO ruling, left on purpose.

**Schema since mig 273 (2026-06-07):** `ref_sku_bom.slot_name` VARCHAR(64) exists; unique key is now **(sku_id, ingredient_raw, source, slot_name)** — the same MI can legitimately appear on two slots of one SKU (e.g. label-beer -B: bottle-label ×24 + box_label ×1). All 3 compiler INSERT paths write slot_name. `ref_sku_packaging_choices.qty_per_unit` is DECIMAL(14,6) (fractional template qtys like 0.000930 were silently rounding before).
