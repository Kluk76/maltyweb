## RECOMMENDED ARCHITECTURE
- **Where the code lives:** STRANGLER RULE ‚Üí prefer **maltyweb + MySQL + cron**, NOT a new standalone Node script. The mailbox poller is a Python connector on the VPS (sibling to the BC connectors `scripts/python/ingest_bc_*.py` ‚Äî same cron pattern, same `config/*.env` creds model) that fetches new messages, dispatches to per-sender parsers, and writes `ord_orders` + a raw-email cache table. Parsers: a **PHP or Python per-sender registry mirroring `lib/invoice-parsers/`** ‚Äî `matches(ctx)‚Üíbool` + `parse(body)‚Üí{customer, lines[], requested_date}`. PM leans **Python** (the connector is Python; keeps mailbox+parse+write in one VPS process; the JS invoice-parsers stay JS but the PATTERN is copied, not the code). Validation UI = maltyweb PHP.
- **Email body, not OCR:** email orders are text/HTML bodies (or attached PDF/xlsx). Text bodies ‚Üí regex/structured parse directly (no OCR). PDF/xlsx attachments ‚Üí reuse the OCR/xlsx path. The parser `ctx` = `{fromAddress, subject, bodyText, attachments[]}` (analogue of the invoice `ctx`).
- **Resolution:** customer via `ref_customers` (exact email‚Üícustomer map first ‚Äî store a sender-email‚Üícustomer_id pin, analogous to `ref_mi_aliases`; then name match; else review). SKU via `ref_sku_aliases`/`ref_skus` (reuse the `(Item_No,UoM)` resolver pattern from BC Phase-1). Confidence gate: all lines resolve ‚â• threshold + customer resolved ‚Üí `review_status='auto'`; else `review_status='needs_review'`. Unparseable sender (no `matches`) ‚Üí review bucket with raw body, NEVER guess.
- **Idempotency/dedup:** key on **RFC822 `Message-ID`** (stable per email) ‚Üí `source_ref='email:<message-id>'` on the UNIQUE `source_ref` col. Replies/threads = distinct Message-IDs ‚Üí distinct rows; a forwarded/re-sent same Message-ID = skip. Label processed messages in Gmail (`maltytask/processed`) as a belt-and-suspenders second guard so a label-filter query never re-fetches.
- **Raw store:** new table `doc_email_messages` (message_id UNIQUE, from, subject, received_at, raw_body, parser_matched, parse_status) ‚Äî the append-only raw lane (mirrors `doc_files`/invoice-log). `ord_orders.source_file_id_fk` can point at it (or add `source_email_id_fk`). Refuse-don't-NULL: unresolved line ‚Üí review row, never a NULL sku_id_fk ord_order_line.
