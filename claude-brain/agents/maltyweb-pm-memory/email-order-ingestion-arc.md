PM leans **Python** (the connector is Python; keeps mailbox+parse+write in one VPS process; the JS invoice-parsers stay JS but the PATTERN is copied, not the code). Validation UI = maltyweb PHP.
- **Email body, not OCR:** email orders are text/HTML bodies (or attached PDF/xlsx). Text bodies тЖТ regex/structured parse directly (no OCR). PDF/xlsx attachments тЖТ reuse the OCR/xlsx path. The parser `ctx` = `{fromAddress, subject, bodyText, attachments[]}` (analogue of the invoice `ctx`).

- **Idempotency/dedup:** key on **RFC822 `Message-ID`** (stable per email) тЖТ the `doc_email_messages.message_id` UNIQUE col (AS-BUILT) and `source_ref='email:<message-id>'` on `ord_orders`. ## ЁЯЯв PARSER PLAN тАФ THREE-LAYER DETERMINISTIC STACK (Kouros approved 2026-06-15, "parse the BODY without an LLM"). NO LLM, EVER.
Kouros asked "can we parse the email BODY (not just PDF attachments) without an LLM?" тЖТ YES. The registry runs a 3-layer deterministic stack; uncertain тЖТ human, never guessed.
- **Layer 1 тАФ per-sender templates.** Same doctrine as the PDF/invoice parsers, applied to `body_text`/`body_html`: HTML-table DOM walk OR aligned plain-text columns. `matches(ctx)тЖТbool` keyed on from_address/subject; `parse(body)тЖТParsedOrder`. Built against Kouros's real per-sender samples.
- **Layer 2 тАФ GENERIC vocabulary fallback тАФ тЬЕ SHIPPED `0a1f267` (`email_parsers/generic_vocab.py`, registered LAST).** Runs ONLY after all per-sender parsers decline. Vocabulary = **LIVE read** (`load_vocab`) from CANONICAL tables тАФ **`ref_skus`(sku_code, beer_raw), `ref_recipes`(name, recipe_short_name), `ref_beer_types`(beer_name, aliases), `ref_recipe_aliases`, `ref_sku_aliases`** (407 terms on current DB; **NEVER hardcoded** тАФ only docstring examples name beers). Fuzzy = **stdlib `difflib` token-sort ratio тЙе 0.85** (rapidfuzz absent on VPS + WSL тЖТ difflib chosen, ZERO new dep). Deterministic qty/unit grammar (f├╗t/carton/bouteille/canette/pack/palette, "Nx", "N L", FR decimal comma) + FR date-phrase parser honoring `system_settings` dayfirst; **relative-week phrases ("cette semaine") тЖТ None (ambiguous, NOT committed).** Conservative gate: emits a ParsedOrder ONLY for lines mapping to a known SKU with unambiguous qty; zero confident lines тЖТ `None` тЖТ no_match тЖТ review. Vocab loaded once per run into `env` (works in `--dry-run`); declines gracefully if `env.vocab is None` (offline smoke stays green). Tune the grammar against real samples when they arrive.
- **Layer 3 тАФ confidence gate.** Anything below the bar тЖТ the review bucket (the validation-first surface = `doc_email_messages.parse_status` + the P3 UI). "No LLM" = uncertain orders go to a human, never guessed.
- **ЁЯФС Registry dispatch refinement тАФ тЬЕ IMPLEMENTED `0a1f267` (`email_parsers/__init__.py`):** match-AND-parseable-wins. `parse()` returning **`None` = decline, try the next parser** (a per-sender parser that matches the SENDER but can't read THIS body falls through to layer-2); **`raise` = error** (тЖТ parse_status='error'); **all-declined / none-matched = no_match** (тЖТ parse_status='no_match'). **`parse()` signature is now `(ctx, env)`** (env carries vocab + dayfirst); `base.py` + `example_template.py` updated. The example_template is the L1 reference; generic_vocab is the L2 generic parser registered last.
