## 🔴 LOAD-BEARING DISCIPLINE — ADDITIVE COST vs OWNED VOLUME (cardinal correctness rule of this design)
**Cost is additive across BOM lines; VOLUME IS NOT.** Liquid volume is **OWNED by the real CONTAINER LAYER line** (= the SKU's liquid volume). An accessory's `packaging_hl_equivalent` is an **attribution key for consumption, NOT additive liquid** — summing it across all BOM lines would N× (multiply) the volume (container + closure + label + box + … each carry an equiv). The volume_hl on a SKU = the container line's volume ONLY. Any rollup that does `SUM(volume_hl)` across all lines of one SKU is WRONG by construction.

## 🔴 CONTAINER-LAYER VOLUME OWNERSHIP — verified slot taxonomy (corrects the earlier keg→keg_collars proxy; 2026-05-26)
Volume is owned by the real CONTAINER LAYER, and which slot owns it differs by format family. **PM-verified live slot taxonomy** (`ref_packaging_items` slot_name counts): `bottle`(6), `can`(8), `can_lids`(8), `crown_caps`(6), `holder`(2), `keg_collars`(1), `keg_safe`(1), `label`(6), `liner_client`(1), `liner_transport`(1), `outer_box`(9), `outer_tray`(3), `scotch`(3), `sticker`(5), `intercal`(1), `pallet`(2), `renforts_single`(1), `renforts_double`(1).

- **keg** has ONLY `keg_collars` + `keg_safe` slots — **NO keg-container line** (keg is reusable, not a consumable). A collar is an ACCESSORY, not the container.

- **Consequence:** keg/cuv have **NO container BOM line → their volume is a SKU-LEVEL fact via the `v_sku_volume` view** (keg derivable, e.g. EMBF=0.2 HL from KEG_INOX_20; cuv variable/NULL). **The earlier keg→`keg_collars` volume hack is REMOVED from the compile** (committed `a8950ed`, 2026-05-26) — the compile no longer maps keg→keg_collars for volume; collar = accessory, not container.
