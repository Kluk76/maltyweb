### deltaHtml `result.delta` UNIT CONVENTION — VARIES, never blanket-append a suffix
> 🔴 **DURABLE (audited live 2026-06-16, app/kpi-handlers.php + public/js/kpi-charts.js): the shared `deltaHtml(delta, deltaLabel)` renderer (kpi-charts.js ~l.99) prints the delta UNIT-LESS, and `result.delta` semantics VARY across handlers — some are YoY/MoM %, some are ABSOLUTE (HL/CHF/count/units/points).** So a blanket `%` append inside deltaHtml would silently corrupt every absolute-delta tile (it would label an HL or CHF or order-count delta as a percent — the same one-renderer-two-meanings divergence smell as a parallel data store). The unit is currently only HINTED in `delta_label` text (e.g. `vs N-1 YTD` is %; `(%)`, `(pp)`, `pts vs` spell it out; bare `vs mois précédent` is ambiguous and unit-varies). **Examples that REFUTE blanket-%:** `kpi_logi_shipped_orders` (`$shippedOrders-$prior`, **count** of orders, label `vs Nj précédents`); `kpi_*_hl_*` MoM HL variants (`$curHl-$prevHl`, absolute **HL**, no unit in label); COP-section deltas (`$currVal-$prevVal`, absolute **CHF**); units-sold-by-SKU (`$units-$prior`, **units**); util-share (`$pct-$priorPct`, **points**, label `pts vs mois précédent`); naive CHF/HL MoM (`round($curr-$prev,2)`). **% examples:** `kpi_wort_hl_brewed_ytd` (`((currYtd-prevYtd)/prevYtd)*100`, label `vs N-1 YTD` — the tile that surfaced this), sales-revenue/hl-sold MoM (`/prior*100`), the entire energy block (`/prior*100`, labels carry `(%)`). **PM RULING: the minimal correct fix is to THREAD a per-tracker `delta_unit` field on the result payload (default `'%'` since most YoY tiles are %), NOT to hard-code `%` in deltaHtml.** deltaHtml then appends the FR-spaced suffix (`'▲ 8,6 %'` — NBSP/`&thinsp;` + `%` for %; `' HL'`/`' CHF'`/`' cmd'`/`' pts'` etc. for absolutes; empty for unit-less). Wire `delta_unit` in `kpi_empty_result()` default + each handler that returns an absolute delta sets its own. grouped_bar's PER-ROW chip uses its OWN `%` literal (kpi-charts.js ~l.165, already correct) — only the FOOTER scalar goes through deltaHtml, so this change touches the footer/number/sparkline/stacked/line paths, not the grouped per-row chip. Same pattern as the additive per-row `b.unit` override already added for grouped_bar.
