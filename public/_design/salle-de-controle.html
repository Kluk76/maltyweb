Paramètres Conditionnement

Seuils process · Soutirage → Conditionnement · commissioning_settings
Délai soutirage
Seuil d'éligibilité conditionnement
Conditionnement autorisé jours après soutirage (CC)
1j
Un lot peut être conditionné dès qu'il a passé au moins 1 jour en CC/BBT après soutirage. Le formulaire de conditionnement n'affiche que les lots dont le délai est atteint. Ce seuil est configurable ici — la valeur courante de 1 jour est lue depuis commissioning_settings.
Lots éligibles
Logique de gate process
Lots affichés dans le formulaire état éligible uniquement
soutirage ≥ seuil
Le gate compare DATE(submitted_at) du soutirage avec la date courante. Seuls les lots ayant passé le seuil sont proposés dans le menu déroulant — les autres restent invisibles pour l'opérateur jusqu'à éligibilité.
Choix Hors Process — gate d'override
Managers Manager et Admins Admin peuvent conditionner un lot avant que le délai réglementaire soit atteint (urgence commerciale, erreur de planification, test). Ce choix est signalé explicitement dans le formulaire et marque le record avec un flag hors_process = 1 persisté en DB pour audit.

Opérateurs Opérateur ne voient que les lots éligibles — le gate leur est opaque, aucune action hors process n'est exposée dans leur interface.
Implémentation & emplacement admin
Les seuils sont lus depuis la table commissioning_settings (clé packaging_racking_delay_days, valeur courante 1). La page d'administration vit provisoirement sous /admin/settings/packaging.php ; une fois la Salle de contrôle activée en page live, ce panneau Conditionnement en devient l'emplacement canonique — les settings seront éditables directement ici, avec confirmation modale et audit log. Le flag hors_process alimente le journal des lots et sera visible dans la future section QA / Contrôles (Q3).
